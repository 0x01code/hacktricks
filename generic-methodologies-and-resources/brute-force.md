### Crunch
```bash
crunch 4 6 0123456789ABCDEF -o crunch1.txt #From length 4 to 6 using that alphabet
crunch 4 4 -f /usr/share/crunch/charset.lst mixalpha # Only length 4 using charset mixalpha (inside file charset.lst)

@ Lower case alpha characters
, Upper case alpha characters
% Numeric characters
^ Special characters including spac
crunch 6 8 -t ,@@^^%%
```
### Cewl

Το Cewl είναι ένα εργαλείο για τη δημιουργία λεξικών βασισμένων σε λέξεις που βρίσκονται σε έναν στόχο. Αυτό το εργαλείο μπορεί να χρησιμοποιηθεί για την εκμετάλλευση της ανθρώπινης αδυναμίας να δημιουργήσει ισχυρούς κωδικούς πρόσβασης. Το Cewl αναλύει τον στόχο και εξάγει λέξεις από τον ιστότοπο, τα έγγραφα και τα μηνύματα ηλεκτρονικού ταχυδρομείου που σχετίζονται με τον στόχο. Στη συνέχεια, δημιουργεί ένα λεξικό με βάση αυτές τις λέξεις, το οποίο μπορεί να χρησιμοποιηθεί για επιθέσεις με brute force σε συστήματα που χρησιμοποιούν αυτές τις λέξεις ως κωδικούς πρόσβασης.
```bash
cewl example.com -m 5 -w words.txt
```

### AJP

Το AJP (Apache JServ Protocol) είναι ένα πρωτόκολλο επικοινωνίας που χρησιμοποιείται για την επικοινωνία μεταξύ ενός web διακομιστή Apache και ενός εφαρμογής που τρέχει σε έναν Tomcat διακομιστή. Το AJP παρέχει μια αποδοτική και ασφαλή μέθοδο για τη μεταφορά δεδομένων μεταξύ των δύο διακομιστών.

Η επίθεση με χρήση του AJP περιλαμβάνει την εκμετάλλευση ευπάθειών στην υλοποίηση του πρωτοκόλλου για να αποκτήσει μη εξουσιοδοτημένη πρόσβαση στον Tomcat διακομιστή. Με αυτόν τον τρόπο, ο επιτιθέμενος μπορεί να αποκτήσει πρόσβαση σε ευαίσθητες πληροφορίες ή να εκτελέσει κακόβουλο κώδικα στον διακομιστή.

Για να προστατευθείτε από επιθέσεις με χρήση του AJP, μπορείτε να απενεργοποιήσετε την υποστήριξη του πρωτοκόλλου AJP στον Apache διακομιστή σας. Επίσης, είναι σημαντικό να ενημερώνετε τακτικά το λογισμικό σας για να αποφύγετε τυχόν ευπάθειες που μπορεί να εκμεταλλευτούν οι επιτιθέμενοι.
```bash
nmap --script ajp-brute -p 8009 <IP>
```
## AMQP (ActiveMQ, RabbitMQ, Qpid, JORAM και Solace)

Η AMQP (Advanced Message Queuing Protocol) είναι μια πρωτοκολλοποίηση που χρησιμοποιείται για την αποστολή και λήψη μηνυμάτων μεταξύ εφαρμογών. Υποστηρίζεται από διάφορες πλατφόρμες μηνυμάτων, όπως οι ActiveMQ, RabbitMQ, Qpid, JORAM και Solace.

Η επίθεση με brute force σε έναν AMQP broker μπορεί να επιτρέψει σε έναν εισβολέα να αποκτήσει πρόσβαση σε ευαίσθητες πληροφορίες ή να πραγματοποιήσει ανεπιθύμητες ενέργειες. Για να εκτελέσετε μια επίθεση brute force σε έναν AMQP broker, μπορείτε να χρησιμοποιήσετε εργαλεία όπως το Hydra ή το Patator.

Πριν ξεκινήσετε μια επίθεση brute force, είναι σημαντικό να συλλέξετε πληροφορίες για τον στόχο σας, όπως το όνομα χρήστη, τον κατάλογο εφαρμογών και τις προεπιλεγμένες ρυθμίσεις ασφαλείας. Αυτές οι πληροφορίες μπορούν να σας βοηθήσουν να επιλέξετε τις κατάλληλες λέξεις πρόσβασης και να αυξήσετε τις πιθανότητες επιτυχίας της επίθεσης.

Όταν εκτελείτε μια επίθεση brute force, μπορείτε να χρησιμοποιήσετε λεξικά με κοινές λέξεις πρόσβασης, συνδυασμούς από γνωστές πληροφορίες ή ακόμη και λεξικά που προέρχονται από προηγούμενες παραβιάσεις δεδομένων. Επίσης, μπορείτε να δοκιμάσετε διάφορες συνδυαστικές λέξεις πρόσβασης, όπως αριθμούς, γράμματα και ειδικούς χαρακτήρες.

Για να αποφύγετε τον εντοπισμό και τον αποκλεισμό από τον AMQP broker, μπορείτε να χρησιμοποιήσετε την τεχνική του "slow brute force", όπου περιορίζετε τον αριθμό των αιτημάτων που στέλνετε ανά δευτερόλεπτο. Επίσης, μπορείτε να χρησιμοποιήσετε προθεσμίες μεταξύ των αιτημάτων για να μειώσετε τον κίνδυνο ανίχνευσης.

Αν καταφέρετε να αποκτήσετε πρόσβαση σε έναν AMQP broker με επιτυχία, μπορείτε να εκτελέσετε περαιτέρω ενέργειες, όπως την ανάγνωση, την τροποποίηση ή τη διαγραφή μηνυμάτων, την παρακολούθηση της κίνησης των μηνυμάτων ή ακόμη και την εκτέλεση κακόβουλου κώδικα.

Είναι σημαντικό να θυμάστε ότι η εκτέλεση επιθέσεων brute force χωρίς την έγκριση του νόμιμου κατόχου του συστήματος αποτελεί παράνομη δραστηριότητα και μπορεί να έχει νομικές συνέπειες. Πάντα χρησιμοποιείτε αυτές τις τεχνικές με ηθικό τρόπο και μόνο για νόμιμους σκοπούς, όπως τη δοκιμή ασφάλειας του δικού σας συστήματος.
```bash
legba amqp --target localhost:5672 --username admin --password data/passwords.txt [--amql-ssl]
```
### Cassandra

Η Cassandra είναι μια ανοικτού κώδικα κατανεμημένη βάση δεδομένων που χρησιμοποιείται για την αποθήκευση μεγάλου όγκου δεδομένων σε πολλούς κόμβους. Ένας από τους τρόπους που μπορεί να επιτεθεί ένας χάκερ σε ένα σύστημα Cassandra είναι μέσω μιας επίθεσης βίαιης δύναμης.

Η επίθεση βίαιης δύναμης στην Cassandra περιλαμβάνει τη δοκιμή διάφορων συνδυασμών ονομάτων χρηστών και κωδικών πρόσβασης για να ανακαλύψει έγκυρα διαπιστευτήρια. Ο χάκερ μπορεί να χρησιμοποιήσει εργαλεία όπως το Hydra ή το Medusa για να αυτοματοποιήσει τη διαδικασία αυτή.

Για να προστατευθείτε από επιθέσεις βίαιης δύναμης στην Cassandra, μπορείτε να λάβετε τα ακόλουθα μέτρα ασφαλείας:

- Χρησιμοποιήστε ισχυρούς κωδικούς πρόσβασης που περιλαμβάνουν συνδυασμούς γραμμάτων, αριθμών και ειδικών χαρακτήρων.
- Εφαρμόστε πολιτικές κλειδώματος λογαριασμού για να περιορίσετε τον αριθμό των αποτυχημένων προσπαθειών σύνδεσης.
- Χρησιμοποιήστε μηχανισμούς ανίχνευσης και προστασίας από επιθέσεις βίαιης δύναμης, όπως η ανίχνευση ασυνήθιστων προσπαθειών σύνδεσης ή η απαγόρευση αυτόματων εργαλείων επίθεσης.

Είναι σημαντικό να λαμβάνετε υπόψη την ασφάλεια του συστήματός σας και να εφαρμόζετε τις κατάλληλες προφυλάξεις για να αποτρέψετε επιθέσεις βίαιης δύναμης στην Cassandra.
```bash
nmap --script cassandra-brute -p 9160 <IP>
# legba ScyllaDB / Apache Casandra
legba scylla --username cassandra --password wordlists/passwords.txt --target localhost:9042
```
### CouchDB

Το CouchDB είναι ένα ανοιχτού κώδικα σύστημα βάσης δεδομένων που χρησιμοποιεί το πρωτόκολλο HTTP για την αποθήκευση και την ανάκτηση των δεδομένων. Μια από τις πιο γνωστές ευπάθειες του CouchDB είναι η επιθετική δοκιμή κωδικού πρόσβασης (brute force attack) για την ανακάλυψη των σωστών διαπιστευτηρίων. Αυτή η επίθεση εκτελείται με την δοκιμή διάφορων συνδυασμών ονοματεπών και κωδικών πρόσβασης μέχρι να βρεθεί ο σωστός συνδυασμός. 

Για να εκτελέσετε μια επίθεση brute force στο CouchDB, μπορείτε να χρησιμοποιήσετε εργαλεία όπως το Hydra ή το Medusa. Αυτά τα εργαλεία σας επιτρέπουν να ορίσετε μια λίστα με πιθανούς συνδυασμούς ονοματεπών και κωδικών πρόσβασης και να τα δοκιμάσετε αυτόματα εναντίον του CouchDB server. Μπορείτε επίσης να χρησιμοποιήσετε λεξικογραφικές επιθέσεις, όπου χρησιμοποιείτε μια λίστα με συνηθισμένους κωδικούς πρόσβασης για να δοκιμάσετε την αυθεντικοποίηση.

Είναι σημαντικό να σημειωθεί ότι η εκτέλεση μιας επίθεσης brute force είναι παράνομη και απαράδεκτη χωρίς την προηγούμενη άδεια του νόμιμου κατόχου του συστήματος. Πάντα πρέπει να τηρείτε τους νόμους και να χρησιμοποιείτε αυτές τις τεχνικές μόνο για νόμιμους σκοπούς, όπως τη δοκιμή ασφαλείας σε ένα σύστημα που σας ανήκει ή για εκπαιδευτικούς σκοπούς.
```bash
msf> use auxiliary/scanner/couchdb/couchdb_login
hydra -L /usr/share/brutex/wordlists/simple-users.txt -P /usr/share/brutex/wordlists/password.lst localhost -s 5984 http-get /
```
### Καταχώρηση Docker

Η καταχώρηση Docker είναι μια υπηρεσία που χρησιμοποιείται για την αποθήκευση και τη διανομή εικόνων Docker. Μπορεί να χρησιμοποιηθεί για την αποθήκευση ιδιωτικών εικόνων Docker ή για την πρόσβαση σε δημόσιες εικόνες Docker που έχουν δημοσιευθεί από άλλους χρήστες.

Η καταχώρηση Docker χρησιμοποιεί ένα σύστημα αυθεντικοποίησης για να ελέγχει την πρόσβαση στις εικόνες. Οι χρήστες πρέπει να παρέχουν σωστές πιστοποιητικές πληροφορίες για να μπορέσουν να αποκτήσουν πρόσβαση σε μια εικόνα Docker. Αυτό μπορεί να γίνει μέσω ενός χρήστη και ενός κωδικού πρόσβασης ή μέσω ενός κλειδιού πρόσβασης.

Για να προστατευθεί η καταχώρηση Docker από επιθέσεις brute force, μπορούν να ληφθούν μερικά μέτρα ασφαλείας. Ένα από αυτά τα μέτρα είναι η εφαρμογή περιορισμών στον αριθμό των αποτυχημένων προσπαθειών σύνδεσης πριν από τον αποκλεισμό ενός χρήστη. Επίσης, μπορεί να χρησιμοποιηθεί η διπλή επαλήθευση για να επιβεβαιωθεί η ταυτότητα του χρήστη πριν του επιτραπεί η πρόσβαση.

Επιπλέον, είναι σημαντικό να χρησιμοποιούνται ισχυροί κωδικοί πρόσβασης και να ενημερώνονται τακτικά. Επίσης, μπορεί να εφαρμοστεί η πολιτική του "λουκέτου" για να αποτραπεί η πρόσβαση μετά από μια συγκεκριμένη περίοδο αδράνειας.

Τέλος, η παρακολούθηση των αρχείων καταγραφής είναι σημαντική για την ανίχνευση ενδεχόμενων επιθέσεων brute force. Οι αποτυχημένες προσπάθειες σύνδεσης και οι αποκλεισμένοι χρήστες πρέπει να καταγράφονται και να αναλύονται για την αντιμετώπιση τυχόν προβλημάτων ασφαλείας.
```
hydra -L /usr/share/brutex/wordlists/simple-users.txt  -P /usr/share/brutex/wordlists/password.lst 10.10.10.10 -s 5000 https-get /v2/
```
### Elasticsearch

Το Elasticsearch είναι μια ανοιχτού κώδικα πλατφόρμα αναζήτησης και ανάλυσης δεδομένων. Χρησιμοποιείται ευρέως για την αποθήκευση, αναζήτηση και ανάλυση μεγάλων όγκων δεδομένων. Η ασφάλεια του Elasticsearch είναι κρίσιμη για την προστασία των δεδομένων σας.

#### Επίθεση με βίαιη δύναμη στο Elasticsearch

Η επίθεση με βίαιη δύναμη στο Elasticsearch αποτελεί μια τεχνική που χρησιμοποιείται για να αποκτηθούν πρόσβαση σε ένα σύστημα Elasticsearch με τη δοκιμή διάφορων συνδυασμών ονομάτων χρηστών και κωδικών πρόσβασης. Αυτή η επίθεση μπορεί να είναι αποτελεσματική εάν οι προεπιλεγμένες διαπιστευτήριες πρόσβασης δεν έχουν αλλάξει ή εάν οι κωδικοί πρόσβασης είναι αδύνατον να μαντευτούν.

#### Προστασία από επίθεση με βίαιη δύναμη

Για να προστατευτείτε από μια επίθεση με βίαιη δύναμη στο Elasticsearch, μπορείτε να λάβετε τα παρακάτω μέτρα:

- Αλλάξτε τις προεπιλεγμένες διαπιστευτήριες πρόσβασης.
- Χρησιμοποιήστε ισχυρούς κωδικούς πρόσβασης που περιλαμβάνουν συνδυασμούς γραμμάτων, αριθμών και ειδικών χαρακτήρων.
- Εφαρμόστε περιορισμούς στον αριθμό των αποτυχημένων προσπαθειών σύνδεσης.
- Χρησιμοποιήστε μηχανισμούς ανίχνευσης και προστασίας από επιθέσεις με βίαιη δύναμη.
- Ενημερώστε το Elasticsearch σε τακτικά διαστήματα για να επιλύσετε γνωστά προβλήματα ασφαλείας.

Ακολουθώντας αυτές τις βασικές πρακτικές ασφαλείας, μπορείτε να μειώσετε τον κίνδυνο μιας επίθεσης με βίαιη δύναμη στο Elasticsearch και να προστατεύσετε τα δεδομένα σας.
```

Η βίαιη δύναμη είναι μια τεχνική που προσπαθεί να ανακτήσει τα διαπιστευτήρια πρόσβασης ενός χρήστη δοκιμάζοντας διάφορους συνδυασμούς ονομάτων χρηστών και κωδικών πρόσβασης.

Για να εκτελέσετε μια επίθεση βίαιης δύναμης σε ένα σύστημα MSSQL, μπορείτε να χρησιμοποιήσετε εργαλεία όπως το Hydra ή το Medusa. Αυτά τα εργαλεία σας επιτρέπουν να αυτοματοποιήσετε τη διαδικασία της δοκιμής διαφόρων συνδυασμών ονομάτων χρηστών και κωδικών πρόσβασης.

Όταν εκτελείτε μια επίθεση βίαιης δύναμης, είναι σημαντικό να χρησιμοποιήσετε μια λίστα με κοινούς κωδικούς πρόσβασης και να προσαρμόσετε τις ρυθμίσεις του εργαλείου σας για να αποφύγετε τον αποκλεισμό λόγω πολλαπλών αποτυχημένων προσπαθειών σύνδεσης.

Είναι επίσης σημαντικό να έχετε υπόψη σας ότι η επίθεση βίαιης δύναμης είναι μια παράνομη δραστηριότητα και μπορεί να έχει νομικές συνέπειες. Πριν εκτελέσετε μια επίθεση βίαιης δύναμης, βεβαιωθείτε ότι έχετε την άδεια και τη συγκατάθεση του νόμιμου κατόχου του συστήματος που επιτίθεστε.

### RDP

Το RDP (Remote Desktop Protocol) είναι ένα πρωτόκολλο που χρησιμοποιείται για να απομακρύνει τον έλεγχο ενός υπολογιστή από απόσταση. Με τη χρήση του RDP, μπορείτε να συνδεθείτε σε έναν απομακρυσμένο υπολογιστή και να τον ελέγξετε από τον δικό σας υπολογιστή.

Η επίθεση με brute force στο RDP συνίσταται στη δοκιμή διάφορων συνδυασμών ονομάτων χρηστών και κωδικών πρόσβασης για να αποκτήσετε πρόσβαση σε έναν απομακρυσμένο υπολογιστή. Αυτή η τεχνική είναι αποτελεσματική όταν οι χρήστες χρησιμοποιούν αδύναμους κωδικούς πρόσβασης ή προεπιλεγμένους κωδικούς πρόσβασης.

Για να εκτελέσετε μια επίθεση brute force στο RDP, μπορείτε να χρησιμοποιήσετε εργαλεία όπως το Hydra ή το Medusa. Αυτά τα εργαλεία σας επιτρέπουν να αυτοματοποιήσετε τη διαδικασία της δοκιμής διάφορων συνδυασμών ονομάτων χρηστών και κωδικών πρόσβασης.

Για να προστατευθείτε από επιθέσεις brute force στο RDP, μπορείτε να λάβετε τα εξής μέτρα ασφαλείας:

- Χρησιμοποιήστε ισχυρούς κωδικούς πρόσβασης που περιλαμβάνουν μείγμα κεφαλαίων και πεζών γραμμάτων, αριθμούς και ειδικούς χαρακτήρες.

- Χρησιμοποιήστε ένα εργαλείο παρακολούθησης για να εντοπίσετε απόπειρες brute force και να λάβετε μέτρα αντίδρασης.
- Περιορίστε την πρόσβαση στο RDP μόνο από συγκεκριμένες διευθύνσεις IP ή χρήστες.

Είναι σημαντικό να προστατεύετε το RDP από επιθέσεις brute force, καθώς μια επιτυχημένη επίθεση μπορεί να οδηγήσει σε παραβίαση της απομακρυσμένης μηχανής και στην πρόσβαση σε ευαίσθητα δεδομένα.
```bash

hydra -V -f -L <userslist> -P <passwlist> rdp://<IP>
legba rdp --target localhost:3389 --username admin --password data/passwords.txt [--rdp-domain <RDP_DOMAIN>] [--rdp-ntlm] [--rdp-admin-mode] [--rdp-auto-logon]
```

Τα αρχεία PDF είναι μια δημοφιλής μορφή αρχείου για την αποθήκευση και την κοινοποίηση εγγράφων. Πριν ξεκινήσετε μια επίθεση με βίαιη δύναμη, είναι σημαντικό να έχετε μια λίστα με κοινούς κωδικούς πρόσβασης που μπορεί να χρησιμοποιούνται συχνά. Μπορείτε επίσης να χρησιμοποιήσετε λεξικογραφικές επιθέσεις, όπου χρησιμοποιείτε μια λίστα με κοινές λέξεις ή φράσεις για να δοκιμάσετε ως κωδικούς πρόσβασης.

Είναι επίσης σημαντικό να έχετε υπόψη ότι η επίθεση με βίαιη δύναμη μπορεί να είναι χρονοβόρα και να απαιτεί πολλούς πόρους υπολογιστή. Επίσης, ορισμένοι ιστότοποι ή εφαρμογές μπορεί να έχουν μέτρα ασφαλείας για να αποτρέψουν αυτού του είδους τις επιθέσεις, όπως περιορισμούς στον αριθμό των προσπαθειών κωδικού πρόσβασης ή καθυστερήσεις μεταξύ των προσπαθειών.

Πριν ξεκινήσετε μια επίθεση με βίαιη δύναμη, είναι σημαντικό να εξετάσετε τη νομική πλευρά του θέματος. Σε ορισμένες περιπτώσεις, η επίθεση με βίαιη δύναμη μπορεί να είναι παράνομη και να έχει σοβαρές νομικές συνέπειες. ### Αποκωδικοποίηση NTLM

Η αποκωδικοποίηση NTLM είναι μια τεχνική που χρησιμοποιείται για να αποκτηθεί πρόσβαση σε κωδικοποιημένα NTLM hashes. Αυτή η τεχνική συνήθως χρησιμοποιείται για να αποκτηθούν διαπιστευτήρια χρηστών και να πραγματοποιηθεί εξουσιοδότηση σε ένα σύστημα.

Οι επιθέσεις αποκωδικοποίησης NTLM μπορούν να γίνουν με τη χρήση λεξικογραφικής επίθεσης ή με τη χρήση επίθεσης με βία. Η λεξικογραφική επίθεση περιλαμβάνει τη δοκιμή πολλών κοινών κωδικών πρόσβασης, ενώ η επίθεση με βία περιλαμβάνει τη δοκιμή όλων των πιθανών συνδυασμών χαρακτήρων.

Για να εκτελέσετε μια επίθεση αποκωδικοποίησης NTLM, μπορείτε να χρησιμοποιήσετε εργαλεία όπως το Hashcat ή το John the Ripper. Αυτά τα εργαλεία υποστηρίζουν διάφορους αλγορίθμους κωδικοποίησης NTLM και μπορούν να επιταχύνουν τη διαδικασία αποκωδικοποίησης.

Είναι σημαντικό να σημειωθεί ότι η αποκωδικοποίηση NTLM είναι μια επίθεση που απαιτεί υπολογιστική ισχύ και χρόνο. Ανάλογα με την πολυπλοκότητα του κωδικού πρόσβασης και την ισχύ του συστήματος, η διαδικασία αποκωδικοποίησης μπορεί να διαρκέσει από λίγα λεπτά έως και αρκετές ώρες.

Επίσης, είναι σημαντικό να τηρείτε τους νόμους και τους κανονισμούς που διέπουν την αποκωδικοποίηση και να χρησιμοποιείτε αυτήν την τεχνική μόνο για νόμιμους σκοπούς, όπως τη διεξαγωγή ελέγχων ασφαλείας ή την ανάκτηση χαμένων διαπιστευτηρίων.

### Αποσπασματική Επίθεση Keberoasting

Η αποσπασματική επίθεση Keberoasting είναι μια τεχνική επίθεσης που στοχεύει στην ανάκτηση κρυπτογραφημένων κωδικών πρόσβασης από τον Active Directory (AD). Αυτή η επίθεση εκμεταλλεύεται την αδυναμία του AD να αντισταθεί στην επίθεση brute force για την αποκρυπτογράφηση των κωδικών πρόσβασης που χρησιμοποιούνται για την πιστοποίηση των χρηστών.

Η επίθεση Keberoasting λειτουργεί με τον εξής τρόπο:

1. Ο επιτιθέμενος εντοπίζει λογαριασμούς χρηστών στο AD που χρησιμοποιούν κρυπτογραφημένους κωδικούς πρόσβασης.
2. Ο επιτιθέμενος αντλεί τους κρυπτογραφημένους κωδικούς πρόσβασης από τον AD.
3. Ο επιτιθέμενος εκτελεί μια επίθεση brute force για να αποκρυπτογραφήσει τους κωδικούς πρόσβασης.
4. Όταν ο επιτιθέμενος αποκρυπτογραφήσει έναν κωδικό πρόσβασης, μπορεί να τον χρησιμοποιήσει για να αποκτήσει πρόσβαση σε συστήματα και υπηρεσίες που χρησιμοποιούν αυτόν τον κωδικό πρόσβασης.

Για να προστατευθείτε από την επίθεση Keberoasting, μπορείτε να υιοθετήσετε τις εξής προφυλάξεις:

- Αλλάξτε τους κωδικούς πρόσβασης των λογαριασμών χρηστών που χρησιμοποιούν κρυπτογραφημένους κωδικούς πρόσβασης.
- Χρησιμοποιήστε ισχυρούς κωδικούς πρόσβασης που περιλαμβάνουν μείγμα κεφαλαίων και πεζών γραμμάτων, αριθμούς και ειδικούς χαρακτήρες.
- Εφαρμόστε πολιτικές περιορισμού πρόσβασης για τους λογαριασμούς χρηστών που χρησιμοποιούν κρυπτογραφημένους κωδικούς πρόσβασης.
- Ενημερώστε το AD σε τακτά χρονικά διαστήματα για να αποφευχθεί η αποθήκευση κρυπτογραφημένων κωδικών πρόσβασης για μεγάλο χρονικό διάστημα.

Αν και η επίθεση Keberoasting είναι μια αποτελεσματική τεχνική για την ανάκτηση κρυπτογραφημένων κωδικών πρόσβασης, με την εφαρμογή των παραπάνω προφυλάξεων μπορείτε να προστατευθείτε από αυτήν.

The PGP/GPG private key is a crucial component in the encryption and decryption process of PGP/GPG. It is used to decrypt messages that have been encrypted with the corresponding public key. The private key should always be kept secure and protected, as it grants access to sensitive information.

In order to generate a PGP/GPG private key, you can use tools like GnuPG or Kleopatra. These tools provide a user-friendly interface to generate and manage PGP/GPG keys.

It is important to choose a strong passphrase for your private key, as it adds an extra layer of security. A passphrase should be unique, complex, and not easily guessable.

Once you have generated your private key, it is recommended to back it up in a secure location. This ensures that you can recover your key in case of loss or hardware failure.

Remember to never share your private key with anyone, as it can be used to impersonate you and gain unauthorized access to your encrypted data.

### Hashcat

#### Επιθέσεις με το Hashcat

* **Επίθεση με λίστα λέξεων** (`-a 0`) με κανόνες

Το **Hashcat** ήδη περιέχει ένα **φάκελο με κανόνες**, αλλά μπορείτε να βρείτε [**άλλους ενδιαφέροντες κανόνες εδώ**](https://github.com/kaonashi-passwords/Kaonashi/tree/master/rules).
```
hashcat.exe -a 0 -m 1000 C:\Temp\ntlm.txt .\rockyou.txt -r rules\best64.rule
```
* **Επίθεση με συνδυασμό λιστών λέξεων**

Είναι δυνατόν να συνδυαστούν 2 λίστες λέξεων σε 1 με το hashcat.\
Αν η πρώτη λίστα περιείχε τη λέξη **"hello"** και η δεύτερη περιείχε 2 γραμμές με τις λέξεις **"world"** και **"earth"**. Οι λέξεις `helloworld` και `helloearth` θα δημιουργηθούν.
```bash
# This will combine 2 wordlists
hashcat.exe -a 1 -m 1000 C:\Temp\ntlm.txt .\wordlist1.txt .\wordlist2.txt

# Same attack as before but adding chars in the newly generated words
# In the previous example this will generate:
## hello-world!
## hello-earth!
hashcat.exe -a 1 -m 1000 C:\Temp\ntlm.txt .\wordlist1.txt .\wordlist2.txt -j $- -k $!
```
* **Επίθεση μάσκας** (`-a 3`)
```bash
# Mask attack with simple mask
hashcat.exe -a 3 -m 1000 C:\Temp\ntlm.txt ?u?l?l?l?l?l?l?l?d

hashcat --help #will show the charsets and are as follows
? | Charset
===+=========
l | abcdefghijklmnopqrstuvwxyz
u | ABCDEFGHIJKLMNOPQRSTUVWXYZ
d | 0123456789
h | 0123456789abcdef
H | 0123456789ABCDEF
s | !"#$%&'()*+,-./:;<=>?@[\]^_`{|}~
a | ?l?u?d?s
b | 0x00 - 0xff

# Mask attack declaring custom charset
hashcat.exe -a 3 -m 1000 C:\Temp\ntlm.txt -1 ?d?s ?u?l?l?l?l?l?l?l?1
## -1 ?d?s defines a custom charset (digits and specials).
## ?u?l?l?l?l?l?l?l?1 is the mask, where "?1" is the custom charset.

# Mask attack with variable password length
## Create a file called masks.hcmask with this content:
?d?s,?u?l?l?l?l?1
?d?s,?u?l?l?l?l?l?1
?d?s,?u?l?l?l?l?l?l?1
?d?s,?u?l?l?l?l?l?l?l?1
?d?s,?u?l?l?l?l?l?l?l?l?1
## Use it to crack the password
hashcat.exe -a 3 -m 1000 C:\Temp\ntlm.txt .\masks.hcmask
```
* Επίθεση με χρήση λίστας λέξεων + μάσκα (`-a 6`) / Μάσκα + λίστα λέξεων (`-a 7`)
```bash
# Mask numbers will be appended to each word in the wordlist
hashcat.exe -a 6 -m 1000 C:\Temp\ntlm.txt \wordlist.txt ?d?d?d?d

# Mask numbers will be prepended to each word in the wordlist
hashcat.exe -a 7 -m 1000 C:\Temp\ntlm.txt ?d?d?d?d \wordlist.txt
```
#### Τρόποι του Hashcat

Hashcat υποστηρίζει τέσσερις βασικούς τρόπους επίθεσης:

- Επίθεση με λεξικό (Dictionary Attack): Αυτή η επίθεση χρησιμοποιεί ένα λεξικό με προκαθορισμένες λέξεις για να δοκιμάσει κάθε λέξη ως πιθανό κλειδί.
- Επίθεση με συνδυασμό (Combination Attack): Αυτή η επίθεση δοκιμάζει όλους τους πιθανούς συνδυασμούς από μια λίστα λέξεων.
- Επίθεση με μάσκα (Mask Attack): Αυτή η επίθεση χρησιμοποιεί ένα πρότυπο μάσκας για να δοκιμάσει όλες τις πιθανές παραλλαγές ενός κλειδιού.
- Επίθεση με ανάγνωση από αρχείο (Hybrid Attack): Αυτή η επίθεση συνδυάζει τη χρήση λεξικού με τη χρήση μάσκας για να δοκιμάσει όλες τις πιθανές παραλλαγές.

Κάθε τρόπος επίθεσης έχει τα δικά του πλεονεκτήματα και μειονεκτήματα, και η επιλογή του κατάλληλου τρόπου εξαρτάται από το συγκεκριμένο σενάριο επίθεσης.
```bash
hashcat --example-hashes | grep -B1 -A2 "NTLM"
```
# Αποκρυπτογράφηση Linux Hashes - Αρχείο /etc/shadow

Το αρχείο `/etc/shadow` σε συστήματα Linux περιέχει τους κρυπτογραφημένους κωδικούς πρόσβασης των χρηστών. Αυτοί οι κωδικοί πρόσβασης αποθηκεύονται ως κρυπτογραφημένα hashes και χρησιμοποιούνται για την αυθεντικοποίηση των χρηστών κατά τη διάρκεια της σύνδεσης στο σύστημα.

Για να αποκρυπτογραφήσουμε αυτούς τους κωδικούς πρόσβασης, μπορούμε να χρησιμοποιήσουμε τεχνικές brute force. Η brute force είναι μια μέθοδος που δοκιμάζει όλους τους πιθανούς συνδυασμούς κωδικών πρόσβασης μέχρι να βρεθεί ο σωστός.

Για να εκτελέσουμε μια επίθεση brute force στα hashes του αρχείου `/etc/shadow`, μπορούμε να χρησιμοποιήσουμε εργαλεία όπως το John the Ripper ή το Hashcat. Αυτά τα εργαλεία χρησιμοποιούν λεξικά με κοινούς κωδικούς πρόσβασης ή εκτελούν brute force επιθέσεις με βάση τον καθορισμένο χώρο αναζήτησης.

Είναι σημαντικό να σημειωθεί ότι η εκτέλεση μιας επίθεσης brute force μπορεί να είναι χρονοβόρα και απαιτεί υψηλή υπολογιστική ισχύ. Επίσης, η χρήση brute force για να αποκρυπτογραφήσουμε κωδικούς πρόσβασης είναι παράνομη, εκτός αν έχουμε την άδεια του νόμιμου κατόχου του συστήματος.

Για να προστατεύσουμε τους κωδικούς πρόσβασης από επιθέσεις brute force, μπορούμε να χρησιμοποιήσουμε ισχυρούς κρυπτογραφημένους αλγορίθμους, όπως το bcrypt ή το SHA-512, και να επιβάλουμε περιορισμούς στον αριθμό των αποτυχημένων προσπαθειών σύνδεσης.

Συνοψίζοντας, η αποκρυπτογράφηση Linux hashes από το αρχείο `/etc/shadow` μπορεί να γίνει με τη χρήση τεχνικών brute force, αλλά αυτή η πρακτική είναι παράνομη και απαιτεί ειδικές άδειες. Για να προστατεύσουμε τους κωδικούς πρόσβασης, πρέπει να χρησιμοποιούμε ισχυρούς κρυπτογραφημένους αλγορίθμους και να επιβάλουμε περιορισμούς στις αποτυχημένες προσπάθειες σύνδεσης.
```
500 | md5crypt $1$, MD5(Unix)                          | Operating-Systems
3200 | bcrypt $2*$, Blowfish(Unix)                      | Operating-Systems
7400 | sha256crypt $5$, SHA256(Unix)                    | Operating-Systems
1800 | sha512crypt $6$, SHA512(Unix)                    | Operating-Systems
```
# Αποκωδικοποίηση των Hash των Windows

Η αποκωδικοποίηση των hash των Windows είναι μια συνηθισμένη τεχνική που χρησιμοποιείται στον χώρο του χάκινγκ. Οι hash των Windows αναπαριστούν τους κωδικούς πρόσβασης των χρηστών και αποθηκεύονται στο αρχείο SAM (Security Account Manager) του συστήματος. Αν μπορέσουμε να αποκωδικοποιήσουμε αυτούς τους hash, μπορούμε να αποκτήσουμε πρόσβαση στο σύστημα χωρίς να γνωρίζουμε τον αρχικό κωδικό πρόσβασης.

Υπάρχουν διάφορες μεθόδοι για την αποκωδικοποίηση των hash των Windows, αλλά η πιο δημοφιλής είναι η επίθεση με brute force. Κατά την επίθεση brute force, δοκιμάζουμε διάφορους πιθανούς κωδικούς πρόσβασης μέχρι να βρούμε τον σωστό. Αυτή η διαδικασία μπορεί να είναι χρονοβόρα, αλλά με τη χρήση εξειδικευμένων εργαλείων και τεχνικών, μπορούμε να αυξήσουμε τις πιθανότητες επιτυχίας.

Ένα από τα πιο δημοφιλή εργαλεία για την αποκωδικοποίηση των hash των Windows είναι το John the Ripper. Το John the Ripper είναι ένα εξαιρετικά ισχυρό εργαλείο αποκωδικοποίησης που υποστηρίζει διάφορους αλγορίθμους κρυπτογράφησης και μπορεί να χρησιμοποιηθεί για την αποκωδικοποίηση των hash των Windows.

Επιπλέον, μπορούμε να χρησιμοποιήσουμε λεξικογραφικές επιθέσεις για την αποκωδικοποίηση των hash των Windows. Κατά τη λεξικογραφική επίθεση, χρησιμοποιούμε μια λίστα με κοινούς κωδικούς πρόσβασης και δοκιμάζουμε κάθε κωδικό από αυτήν τη λίστα. Αυτή η μέθοδος μπορεί να είναι πιο γρήγορη από την επίθεση brute force, αλλά εξαρτάται από την ποιότητα της λίστας με τους κωδικούς πρόσβασης.

Τέλος, μπορούμε να χρησιμοποιήσουμε εξειδικευμένα εργαλεία όπως το Hashcat για την αποκωδικοποίηση των hash των Windows. Το Hashcat είναι ένα εργαλείο αποκωδικοποίησης που υποστηρίζει πολλούς αλγορίθμους κρυπτογράφησης και μπορεί να εκτελέσει γρήγορες επιθέσεις brute force και λεξικογραφικές επιθέσεις.

Συνοψίζοντας, η αποκωδικοποίηση των hash των Windows είναι μια σημαντική τεχνική στον χώρο του χάκινγκ. Με τη χρήση εξειδικευμένων εργαλείων και τεχνικών, μπορούμε να αποκτήσουμε πρόσβαση σε συστήματα Windows χωρίς να γνωρίζουμε τον αρχικό κωδικό πρόσβασης.
```
3000 | LM                                               | Operating-Systems
1000 | NTLM                                             | Operating-Systems
```
# Αποκωδικοποίηση Συνηθισμένων Κατακερματισμένων Εφαρμογών

Οι κατακερματισμένες τιμές είναι μια συνηθισμένη μορφή αποθήκευσης κωδικών πρόσβασης σε πολλές εφαρμογές. Αυτό σημαίνει ότι οι πραγματικοί κωδικοί δεν αποθηκεύονται σε καθαρή μορφή, αλλά αντί αυτού, αποθηκεύονται οι κατακερματισμένες τιμές τους. Ο κατακερματισμός είναι μια μορφή κρυπτογράφησης που μετατρέπει τον κωδικό σε μια μοναδική συμβολοσειρά, γνωστή ως κατακερματισμένη τιμή.

Για να αποκωδικοποιήσουμε μια κατακερματισμένη τιμή, χρησιμοποιούμε μια τεχνική που ονομάζεται "brute force". Αυτή η τεχνική περιλαμβάνει τη δοκιμή όλων των πιθανών συνδυασμών κωδικών πρόσβασης μέχρι να βρεθεί ο σωστός. Αυτό μπορεί να γίνει με τη χρήση ειδικών εργαλείων λογισμικού που αυτοματοποιούν τη διαδικασία.

Είναι σημαντικό να σημειωθεί ότι η αποκωδικοποίηση κατακερματισμένων τιμών μπορεί να είναι χρονοβόρα και απαιτεί υπολογιστική ισχύ. Επίσης, η επιτυχία εξαρτάται από την ποιότητα του κατακερματισμού και την πολυπλοκότητα του κωδικού πρόσβασης.

Παρακάτω παρέχονται ορισμένες συνηθισμένες μεθόδοι αποκωδικοποίησης κατακερματισμένων τιμών:

## Επίθεση με λεξικό

Η επίθεση με λεξικό εμπλέκει τη χρήση ενός λεξικού με προκαθορισμένες λέξεις ή συνδυασμούς λέξεων για να δοκιμάσει ως κωδικούς πρόσβασης. Αυτή η μέθοδος είναι αποτελεσματική όταν ο κωδικός πρόσβασης είναι αδύναμος ή χρησιμοποιεί συνηθισμένες λέξεις.

## Επίθεση με μάσκα

Η επίθεση με μάσκα εμπλέκει τη δημιουργία ενός προτύπου με το οποίο ορίζονται οι πιθανοί χαρακτήρες και οι θέσεις τους στον κωδικό πρόσβασης. Αυτή η μέθοδος είναι χρήσιμη όταν γνωρίζουμε ορισμένα χαρακτηριστικά του κωδικού πρόσβασης, όπως το μήκος ή οι συγκεκριμένοι χαρακτήρες που περιλαμβάνει.

## Επίθεση με γεννήτρια

Η επίθεση με γεννήτρια εμπλέκει τη χρήση ενός προγράμματος που δημιουργεί αυτόματα πιθανούς κωδικούς πρόσβασης με βάση καθορισμένους κανόνες. Αυτή η μέθοδος είναι χρήσιμη όταν δεν έχουμε πληροφορίες για τον κωδικό πρόσβασης, αλλά γνωρίζουμε τους κανόνες που ακολουθεί ο χρήστης για τη δημιουργία του.

Αυτές είναι μερικές από τις συνηθισμένες μεθόδους αποκωδικοποίησης κατακερματισμένων τιμών που μπορούν να χρησιμοποιηθούν για την αποκρυπτογράφηση κωδικών πρόσβασης εφαρμογών.
