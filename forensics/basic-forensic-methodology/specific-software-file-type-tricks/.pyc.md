## Od .pyc do python koda

Za **.pyc** podatke ("kompajlirani" python), trebali biste poƒçeti poku≈°avati **izvuƒái** **originalni** **python** **kod**:
```bash
uncompyle6 binary.pyc  > decompiled.py
```
**Proverite** da li binarna datoteka ima **ekstenziju** "**.pyc**" (ako nema, uncompyle6 neƒáe raditi)

Prilikom izvr≈°avanja **uncompyle6** mo≈æda ƒáete naiƒái na **sledeƒáe gre≈°ke**:

### Gre≈°ka: Nepoznat broj magije 227
```bash
/kali/.local/bin/uncompyle6 /tmp/binary.pyc
Unknown magic number 227 in /tmp/binary.pyc
```
Da biste to popravili, trebate **dodati ispravan magiƒçni broj** na poƒçetak generisanog fajla.

**Magiƒçni brojevi variraju sa verzijom Python-a**, da biste dobili magiƒçni broj za **Python 3.8**, trebate **otvoriti Python 3.8** terminal i izvr≈°iti:
```
>> import imp
>> imp.get_magic().hex()
'550d0d0a'
```
**Magiƒçni broj** u ovom sluƒçaju za python3.8 je **`0x550d0d0a`**, zatim, da biste ispravili ovu gre≈°ku, trebaƒáe vam **dodati** na **poƒçetak** **.pyc fajla** sledeƒáe bajtove: `0x0d550a0d000000000000000000000000`

**Kada** ste **dodali** taj magiƒçni zaglavlje, **gre≈°ka bi trebala biti ispravljena.**

Ovako ƒáe izgledati ispravno dodato **.pyc python3.8 magiƒçno zaglavlje**:
```bash
hexdump 'binary.pyc' | head
0000000 0d55 0a0d 0000 0000 0000 0000 0000 0000
0000010 00e3 0000 0000 0000 0000 0000 0000 0000
0000020 0700 0000 4000 0000 7300 0132 0000 0064
0000030 0164 006c 005a 0064 0164 016c 015a 0064
```
### Gre≈°ka: Dekompilacija generiƒçkih gre≈°aka

**Drugaƒçije gre≈°ke** poput: `class 'AssertionError'>; co_code should be one of the types (<class 'str'>, <class 'bytes'>, <class 'list'>, <class 'tuple'>); is type <class 'NoneType'>` mogu se pojaviti.

Ovo verovatno znaƒçi da **nisu pravilno dodati** magiƒçni brojevi ili da niste **koristili ispravan magiƒçni broj**, pa se uverite da koristite ispravan (ili poku≈°ajte sa novim).

Proverite prethodnu dokumentaciju o gre≈°kama.

## Automatski alat

Alat **[python-exe-unpacker](https://github.com/countercept/python-exe-unpacker)** predstavlja kombinaciju nekoliko alata dostupnih zajednici, namenjenih istra≈æivaƒçima za raspakivanje i dekompilaciju izvr≈°nih datoteka napisanih u Pythonu, posebno onih kreiranih sa py2exe i pyinstaller. Ukljuƒçuje YARA pravila za identifikaciju da li je izvr≈°na datoteka bazirana na Pythonu i potvrƒëuje alat za kreiranje.

### ImportError: Ime datoteke: 'unpacked/malware\_3.exe/**pycache**/archive.cpython-35.pyc' ne postoji

ƒåest problem koji se javlja ukljuƒçuje nepotpunu Python bajtkod datoteku koja proizlazi iz procesa raspakivanja sa unpy2exe ili pyinstxtractor, a zatim **nije prepoznata od strane uncompyle6 zbog nedostajuƒáeg broja verzije Python bajtkoda**. Da bi se ovo re≈°ilo, dodata je opcija prepend, koja dodaje potreban broj verzije Python bajtkoda, olak≈°avajuƒái proces dekompilacije.

Primer problema:

## Analiza Python asembliranja

Ako niste uspeli da izvuƒçete "originalni" Python kod koristeƒái prethodne korake, mo≈æete poku≈°ati da izvuƒçete asembliranje (ali to nije veoma opisno, pa poku≈°ajte ponovo da izvuƒçete originalni kod). Na [ovom linku](https://bits.theorem.co/protecting-a-python-codebase/) sam prona≈°ao veoma jednostavan kod za rastavljanje _.pyc_ binarnog fajla (sreƒáno sa razumevanjem toka koda). Ako je _.pyc_ fajl iz Python 2, koristite python2:
```bash
>>> import dis
>>> import marshal
>>> import struct
>>> import imp
>>>
>>> with open('hello.pyc', 'r') as f:  # Read the binary file
...     magic = f.read(4)
...     timestamp = f.read(4)
...     code = f.read()
...
>>>
>>> # Unpack the structured content and un-marshal the code
>>> magic = struct.unpack('<H', magic[:2])
>>> timestamp = struct.unpack('<I', timestamp)
>>> code = marshal.loads(code)
>>> magic, timestamp, code
((62211,), (1425911959,), <code object <module> at 0x7fd54f90d5b0, file "hello.py", line 1>)
>>>
>>> # Verify if the magic number corresponds with the current python version
>>> struct.unpack('<H', imp.get_magic()[:2]) == magic
True
>>>
>>> # Disassemble the code object
>>> dis.disassemble(code)
1           0 LOAD_CONST               0 (<code object hello_world at 0x7f31b7240eb0, file "hello.py", line 1>)
3 MAKE_FUNCTION            0
6 STORE_NAME               0 (hello_world)
9 LOAD_CONST               1 (None)
12 RETURN_VALUE
>>>
>>> # Also disassemble that const being loaded (our function)
>>> dis.disassemble(code.co_consts[0])
2           0 LOAD_CONST               1 ('Hello  {0}')
3 LOAD_ATTR                0 (format)
6 LOAD_FAST                0 (name)
9 CALL_FUNCTION            1
12 PRINT_ITEM
13 PRINT_NEWLINE
14 LOAD_CONST               0 (None)
17 RETURN_VALUE
```
## Python u izvr≈°nu datoteku

Da bismo zapoƒçeli, pokazaƒáemo vam kako se payloadi mogu kompajlirati pomoƒáu py2exe i PyInstaller alata.

### Kako kreirati payload pomoƒáu py2exe:

1. Instalirajte py2exe paket sa [http://www.py2exe.org/](http://www.py2exe.org)
2. Za payload (u ovom sluƒçaju, nazvaƒáemo ga hello.py), koristite skriptu kao ≈°to je prikazano na slici 1. Opcija "bundle\_files" sa vredno≈°ƒáu 1 ƒáe sve, ukljuƒçujuƒái Python interpreter, spojiti u jednu izvr≈°nu datoteku.
3. Kada je skripta spremna, izdajemo komandu "python setup.py py2exe". Ovo ƒáe kreirati izvr≈°nu datoteku, kao ≈°to je prikazano na slici 2.
```python
from distutils.core import setup
import py2exe, sys, os

sys.argv.append('py2exe')

setup(
options = {'py2exe': {'bundle_files': 1}},
#windows = [{'script': "hello.py"}],
console = [{'script': "hello.py"}],
zipfile = None,
)
```
